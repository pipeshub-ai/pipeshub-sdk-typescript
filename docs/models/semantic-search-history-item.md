# SemanticSearchHistoryItem

One persisted search row as returned by `GET /search`. Mirrors the
persisted search document except `citationIds` is an array of
ObjectId strings (citations are not populated on the list endpoint,
unlike `GET /search/{searchId}`).

`shareLink` is absent from the JSON when the search has not been
shared. `archivedBy` is `null` on rows that were previously archived
and then unarchived, and absent on rows that have never been
archived. The list endpoint already filters out currently-archived
rows, so a string user-id value never appears here in practice.


## Example Usage

```typescript
import { SemanticSearchHistoryItem } from "@pipeshub-ai/sdk/models";

let value: SemanticSearchHistoryItem = {
  id: "<value>",
  v: 444266,
  query: "<value>",
  limit: 710619,
  orgId: "<value>",
  userId: "<value>",
  citationIds: [
    "<value 1>",
    "<value 2>",
    "<value 3>",
  ],
  records: {},
  isShared: false,
  sharedWith: [],
  isArchived: false,
  createdAt: new Date("2026-07-20T11:51:28.760Z"),
  updatedAt: new Date("2024-07-28T21:42:34.861Z"),
};
```
